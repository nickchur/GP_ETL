# GP_ETL — система ETL для Greenplum/PostgreSQL (Сбербанк HR-аналитика)

Репозиторий содержит SQL-компоненты ETL-системы для загрузки и трансформации HR-данных в Greenplum. Данные проходят через четыре слоя: **PXF → DIA → STG → CDM**. Система реализует super workflow engine (SWF) для оркестрации параллельных загрузок, фреймворк Data Quality для контроля качества и инфраструктуру логирования и почтовых уведомлений.

## Архитектура слоёв данных

```
Источники (SAP HR, Pulse, Jira, UDL, PXF)
    ↓
DIA  — промежуточный слой (сырые данные из источников)
    ↓
STG  — staging-слой (нормализованные, очищенные данные)
    ↓
CDM  — витрины данных (для аналитиков)
```

## Быстрый старт

Для понимания системы рекомендуется изучать компоненты в следующем порядке:
1. `tables/` — структуры данных (что хранится)
2. `utils/` — вспомогательные функции (базовый слой)
3. `etl_core/` — движки загрузки (как работает ETL)
4. `log/` и `dq/` — мониторинг и контроль качества
5. `srv_wf/` — конкретные пайплайны для HR-сущностей
